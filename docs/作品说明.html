Cipher Garden

作者：康升阳 · 上海视觉艺术学院 23 级时尚设计与传播专业

一句话介绍

这是一个让朋友与创作者通过种花、解谜、扫码分享，体验好奇、期待与揭晓惊喜的网页互动艺术作品。

如何体验

方式一 · 在线体验

在浏览器中打开 GitHub Pages 地址，无需安装任何软件，即可创作或解谜花园。

方式二 · 本地运行

下载项目后双击 start.bat，自动启动本地 HTTP 服务并打开浏览器。因使用 ES Module，不可直接双击 index.html 打开。

交互说明

  • 创作花园：在 5×5 花圃中选择 emoji、旋转角度（0°/90°/180°/270°）与花色，点击格子种植；再次点击相同组合可清除。
  • 密语类型：支持文字密语、照片密语（相机拍摄或相册导入）、文字+照片组合；可选录制最长 3 秒的短视频密语。
  • 分享传播：生成分享链接后，可复制文本发给好友，或使用「微信扫码打开」二维码——扫一扫即可在内置浏览器中打开完整谜题。
  • 解谜花园：打开分享链接后，观察目标花圃（parametric SVG 花朵，不显示 emoji 答案），在猜测区为每格选择 emoji / 旋转 / 花色。
  • Wordle 式反馈：共 3 次猜测机会，每格反馈「完全匹配」「emoji 对但旋转/色错」「emoji 在别格」或「不匹配」。
  • 揭晓时刻：全部猜对后触发绽放动画，文字、照片或短视频密语淡入揭晓——完成从好奇到惊喜的仪式性体验。
  • 草稿保存：创作过程中的花园状态自动保存至浏览器 localStorage，刷新页面不丢失。

艺术理念

「异步密语花园」将编码、社交分享与揭晓仪式编织为一次完整的情感传递体验。创作者用 emoji、旋转与花色在 5×5 花圃中「种植」视觉密码，将文字、照片或短视频藏入花朵深处——密语不会立刻呈现，而是等待解谜者通过观察、记忆与推理逐步逼近真相。分享链接或二维码成为跨越时空的「花种」：发送者与接收者不必同时在线，好奇与期待在异步的等待中慢慢生长。三次猜测机会与 Wordle 式反馈，既保留游戏的挑战感，又让揭晓前的悬念层层累积；当最后一格对齐、花朵绽放、密语淡入的那一刻，解谜从智力活动升华为一种小型仪式——关于信任、用心与被看见的温柔回响。作品以纯前端网页为载体，让数字时代的秘密交换回归手作般的温度与花园般的诗意。

技术实现

HTML / CSS / JavaScript ES Modules lz-string 压缩 QR 二维码 相机 API Catbox 图床 Parametric SVG
  • 纯前端架构：Vanilla HTML / CSS / JavaScript（ES Modules），无后端依赖，谜题状态压缩编码在 URL 哈希段中，便于链接分享。
  • 数据压缩：使用 lz-string 库对谜题 JSON 进行压缩，v2 紧凑格式（短键名、裸 base64）比旧版多容纳约 15–30% 照片数据。
  • 二维码分享：集成 qrcode 库生成微信可扫二维码；支持「微信分享」（≤2048 字符）与「高清扫码」（≤3100 字符）两种模式。
  • 相机与媒体：调用浏览器 MediaDevices API 实现拍照与 3 秒短视频录制；照片导入时轻度压缩（最长边约 1920px），生成链接时按分享模式进一步优化。
  • 云端托管：大照片无法嵌入链接时，自动上传至 litterbox.catbox.moe 临时图床（24 小时有效），链接中仅保存短 URL，确保二维码始终可扫。
  • 视觉生成：Parametric SVG 花朵由 emoji + 旋转 + 花色哈希推导花瓣数、形状与渐变，解谜界面只展示抽象花朵而非明文 emoji。
  • 解谜逻辑：Wordle 式逐格反馈判定（完全匹配 / emoji 对旋转色错 / emoji 在别格 / 不匹配），三次机会后揭晓或失败。
